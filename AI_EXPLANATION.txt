شرح مفصل للـ AI Medical Intelligence في مشروع MediFlow
================================================================

================================================================
1. شنو هو الـ AI اللي عملناه؟
================================================================

عملنا نظام ذكاء اصطناعي للـ Super Admin يقدر يطلب تحليل طبي
للبيانات الحقيقية من قاعدة البيانات.

الفكرة البسيطة:
- الـ Super Admin يضغط زر
- الـ Backend يجمع البيانات من MongoDB
- يبعثها لـ Groq AI (مجاني)
- Groq يولد تحليل طبي بالفرنسية
- يرجع للـ Frontend ويعرضه

================================================================
2. الملفات اللي عملناها
================================================================

BACKEND:
--------
backend/src/modules/ai/ai.service.ts
  - الـ Service الرئيسي
  - يجمع البيانات من DB
  - يبني الـ prompt
  - يتصل بـ Groq API
  - يرجع النتيجة

backend/src/modules/ai/ai.controller.ts
  - Endpoint واحد: POST /ai/report
  - محمي بـ @Permissions('*') = super admin فقط
  - يستقبل { type: string }

backend/src/modules/ai/ai.module.ts
  - يسجل الـ models اللي يحتاجها الـ service
  - مستقل تماماً عن باقي الـ modules

FRONTEND:
---------
frontend/src/app/pages/super-admin/ai-intelligence/
  - ai-intelligence.component.ts  → الـ logic
  - ai-intelligence.component.html → الـ UI
  - ai-intelligence.component.scss → الـ styles

3. كيفاش يشتغل الكود خطوة بخطوة
================================================================

STEP 1: المستخدم يضغط زر في الـ Frontend
-----------------------------------------
الـ Frontend عنده 4 أزرار:
  - "Rapport mensuel"    → type = 'monthly'
  - "Patients à risque"  → type = 'risk'
  - "Coordinateurs"      → type = 'coordinators'
  - "Anomalies"          → type = 'anomalies'

لما يضغط زر، يبعث:
  POST /ai/report
  Body: { "type": "monthly" }
  Authorization: Bearer <token>

STEP 2: الـ Controller يستقبل الطلب
-------------------------------------
ai.controller.ts يتحقق:
  1. هل المستخدم مسجل دخول؟ (JwtAuthGuard)
  2. هل عنده permission '*'؟ (super admin فقط)
  3. إذا OK → يمرر للـ Service

STEP 3: الـ Service يجمع البيانات من DB
-----------------------------------------
ai.service.ts يعمل:

  أ) يجيب role IDs:
     - يبحث عن role اسمه 'patient' → يجيب الـ _id
     - يبحث عن role اسمه 'coordinator' → يجيب الـ _id

  ب) يجيب البيانات بالتوازي (Promise.all):
     - كل المرضى غير المؤرشفين
     - كل الـ coordinators غير المؤرشفين
     - كل الـ reminders

  ج) يحسب vitals وsymptoms اليوم:
     - يجيب vitals من اليوم فقط (recordedAt >= today)
     - يجيب symptoms من اليوم فقط (reportedAt >= today)

  د) يحسب الإحصائيات:
     - okCount: عدد المرضى اللي قدموا vitals + symptoms اليوم
     - noDataCount: عدد المرضى بدون أي بيانات اليوم
     - complianceRate: نسبة الامتثال (okCount / totalPatients * 100)
     - sentReminders: عدد الـ reminders المرسلة
     - monthReminders: عدد الـ reminders هذا الشهر

STEP 4: يبني الـ Prompt
------------------------
يبني نص يحتوي على البيانات الحقيقية:

  "Statistiques plateforme MediFlow:
   - Patients total: 24
   - Conformes aujourd'hui: 0
   - Sans données aujourd'hui: 24
   - Taux de compliance: 0%
   - Coordinateurs: 21
   - Reminders envoyés: 33
   - Reminders ce mois: 42
   - Date: 20/04/2026"

ثم يضيف التعليمات حسب نوع التقرير:
  - monthly:      "Génère un rapport mensuel..."
  - risk:         "Identifie les patterns de risque..."
  - coordinators: "Analyse la performance des coordinateurs..."
  - anomalies:    "Analyse les anomalies..."

STEP 5: يتصل بـ Groq API
--------------------------
يبعث الـ prompt لـ:
  POST https://api.groq.com/openai/v1/chat/completions

مع:
  - model: 'llama-3.1-8b-instant' (مجاني وسريع)
  - system: "You are a medical platform analyst. Always respond in French."
  - max_tokens: 400
  - temperature: 0.4 (متوازن بين الإبداع والدقة)

STEP 6: يرجع النتيجة
---------------------
الـ Service يرجع:
  {
    type: "monthly",
    response: "Rapport mensuel: Ce mois, 24 patients ont été suivis...",
    generatedAt: "2026-04-20T17:20:36Z",
    data: {
      totalPatients: 24,
      totalCoordinators: 21,
      okCount: 0,
      noDataCount: 24,
      complianceRate: 0,
      sentReminders: 33,
      monthReminders: 42
    }
  }

STEP 7: الـ Frontend يعرض النتيجة
-----------------------------------
  - يعرض النص في card بنفسجي
  - يعرض الإحصائيات في 4 boxes
  - يحفظ في الـ history (آخر 5 تقارير)

================================================================
4. الفرق بين الـ AI اللي عملناه والـ rule-based
================================================================

Rule-based (auditor insights):
  - قواعد ثابتة مبرمجة
  - مثال: "إذا compliance < 30% → اعرض تحذير"
  - النتيجة دائماً نفسها لنفس البيانات
  - لا يفهم السياق

AI (super admin):
  - يفهم البيانات ويحللها
  - يعطي توصيات مخصصة
  - يكتب تقرير طبيعي بالفرنسية
  - يقدر يلاحظ أنماط غير متوقعة

================================================================
5. الأمان
================================================================

  - Endpoint محمي بـ JWT (لازم تكون مسجل دخول)
  - @Permissions('*') = super admin فقط
  - الـ Groq API key في الكود مباشرة (للـ dev فقط)
  - في الـ production لازم تحطها في .env

  - تضيف export PDF
  - تضيف AI للـ auditor بنفس الطريقة
